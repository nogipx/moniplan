# Реализация Telegram-бота для лицензирования

## Обзор структуры проекта

Проект `licensify_tg_bot` реализует Telegram-бота для выдачи лицензий приложениям согласно описанной схеме лицензирования. Основной функционал включает:

1. Прием и расшифровку запросов лицензий (`.mlr` файлы)
2. Создание и подписание лицензионных файлов
3. Управление лицензиями (статистика, отзыв)
4. Интеграцию с Dart-скриптами для криптографических операций

### Структура файлов

```
licensify_tg_bot/
├── bot/
│   ├── __init__.py
│   └── bot.py            # Основной код Telegram-бота
├── utils/
│   ├── __init__.py
│   ├── config.py         # Настройки из переменных окружения
│   ├── crypto.py         # Криптографические утилиты
│   ├── license_store.py  # Хранилище лицензий
│   └── models.py         # Модели данных (лицензия, запрос)
├── dart_scripts/
│   ├── pubspec.yaml      # Зависимости Dart
│   ├── decrypt.dart      # Скрипт для расшифровки запросов
│   ├── sign.dart         # Скрипт для подписи лицензий
│   └── verify.dart       # Скрипт для проверки подписей
├── .env.example          # Пример файла конфигурации
├── README.md             # Документация
├── requirements.txt      # Зависимости Python
└── run.py               # Скрипт для запуска бота
```

## Процесс работы

### 1. Прием запроса лицензии

1. Пользователь отправляет боту файл запроса лицензии (`.mlr`)
2. Бот расшифровывает запрос с использованием приватного ключа
3. Проверяется срок действия запроса
4. Проверяется наличие существующих лицензий для устройства

### 2. Выдача лицензии

1. Бот запрашивает у пользователя тип лицензии и срок действия
2. Создается лицензия с соответствующими параметрами
3. Лицензия подписывается с использованием приватного ключа
4. Подписанная лицензия отправляется пользователю в виде JSON-файла
5. Информация о выданной лицензии записывается в хранилище

### 3. Управление лицензиями

Администраторы бота имеют доступ к дополнительным функциям:
- Просмотр статистики по выданным лицензиям
- Отзыв лицензий по их идентификатору

## Криптографические механизмы

В проекте предусмотрены два варианта работы с криптографией:

1. **Встроенные механизмы Python** - используют библиотеку `cryptography` для базовых операций
2. **Dart-скрипты** - запускаются как внешние процессы для совместимости с криптографией Flutter-приложения

### Алгоритмы

- **Шифрование/дешифрование**: ECDSA / RSA-OAEP
- **Подпись**: ECDSA с SHA-512 / RSA-PKCS1 с SHA-512
- **Хеширование**: SHA-512

## Безопасность

- Приватный ключ хранится только на сервере и не передается клиентам
- Каждая лицензия привязана к уникальному хешу устройства
- Подписи предотвращают подделку или изменение лицензий
- Лицензии могут иметь ограниченный срок действия

## Запуск и настройка

Для запуска бота необходимо:

1. Создать файл `.env` на основе `.env.example` с настройками бота
2. Сгенерировать ключи ECDSA (скрипт `run.py` может сделать это автоматически)
3. Установить зависимости Python из `requirements.txt`
4. Установить Dart SDK для работы скриптов
5. Запустить бота командой `python run.py`

## Ограничения и примечания

- Dart-скрипты используют упрощенную реализацию для демонстрации и могут требовать доработки
- В реальном приложении рекомендуется использовать более надежные механизмы хранения лицензий (база данных)
- Для промышленного использования следует добавить дополнительные проверки безопасности
