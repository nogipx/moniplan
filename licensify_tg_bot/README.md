# Licensify Telegram Bot

Телеграм бот для обработки запросов лицензий и генерации лицензионных файлов для приложений.

## Особенности

- Прием и обработка запросов лицензий (`.mlr` файлы)
- Генерация лицензионных файлов с криптографической защитой
- Валидация сигнатур и проверка срока действия запросов
- Интеграция с Dart-скриптами для переиспользования криптографической логики

## Установка

1. Установить зависимости Python:
```bash
pip install -r requirements.txt
```

2. Установить необходимые зависимости Dart (если нужно):
```bash
cd dart_scripts
dart pub get
```

3. Настроить конфигурацию:
- Создать файл `.env` с переменными окружения (на основе `.env.example`)
- Указать токен бота и путь к ключам шифрования

## Запуск

```bash
python -m bot.bot
```

## Структура проекта

- `bot/` - Исходный код Telegram бота
- `utils/` - Утилиты Python для работы с лицензиями
- `dart_scripts/` - Dart-скрипты для криптографических операций 

## Конфигурация типов лицензий

Telegram-бот поддерживает настройку типов лицензий и сроков их действия без необходимости изменения кода. Для этого используется JSON-файл конфигурации, путь к которому задается в `.env` файле через переменную `LICENSE_CONFIG_PATH`.

### Формат конфигурационного файла

```json
{
  "license_types": [
    {
      "id": "standard",
      "name": "Стандартная",
      "description": "Базовая лицензия с основным функционалом",
      "features": {
        "premium_features": false,
        "business_features": false
      },
      "order": 1
    },
    {
      "id": "premium",
      "name": "Премиум", 
      "description": "Расширенная лицензия",
      "features": {
        "premium_features": true,
        "business_features": false,
        "cloud_storage": true
      },
      "order": 2
    }
  ],
  "durations": [
    {
      "id": "30",
      "name": "1 месяц",
      "days": 30,
      "order": 1
    },
    {
      "id": "unlimited",
      "name": "Бессрочно",
      "days": null,
      "order": 2
    }
  ]
}
```

### Параметры типов лицензий

- `id` - уникальный идентификатор типа лицензии
- `name` - название типа лицензии, отображаемое в интерфейсе бота
- `description` - описание типа лицензии (необязательно)
- `features` - объект с ключами-фичами и их значениями (true/false)
- `order` - порядок отображения в интерфейсе бота

### Параметры сроков действия

- `id` - уникальный идентификатор срока действия
- `name` - название срока действия, отображаемое в интерфейсе бота
- `days` - количество дней действия лицензии (null для бессрочной)
- `order` - порядок отображения в интерфейсе бота

### Пример использования конфигурации

Для использования другого файла конфигурации укажите его путь в `.env` файле:

```
LICENSE_CONFIG_PATH=./data/custom_license_types.json
```

При первом запуске, если файл конфигурации не найден, будет создана конфигурация по умолчанию.
